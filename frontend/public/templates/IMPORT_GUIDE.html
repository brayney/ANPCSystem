Equipment Data Import Guide with Required Fields

🏗️ Cranes Template
File: cranes-template.csv | Required Fields: equipmentNo, craneModel, yearModel, capacity
Field Name Type Description Example
equipmentNo String (Unique) Unique equipment identifier CRANE-001
craneModel String Model name/number of the crane Liebherr LTM 1100
yearModel Number Year of manufacture 2020
capacity String Lifting capacity (e.g., "150T", "200T MAX") 150T
location String Current storage/operational location RAG YARD
client String Assigned client or "-" if none -
status String Available / In Use / Under Maintenance / Out of Yard Available
condition String OK / NOT OK / For Repair / Unknown OK

Notes:

  • equipmentNo must be unique and is the primary identifier
  • supervisor field has been removed from import
  • Default location is "RAG YARD" if not specified
  • Default status is "Available"

📋 General Import Guidelines

CSV Format Requirements:

  • Use UTF-8 encoding for proper character support
  • First row must contain field headers (column names)
  • Do not include row numbers or empty columns
  • Empty cells are allowed; defaults will be applied based on field type
  • Use commas to separate fields; quote values containing commas: "Value, with comma"

Status Values (Case-Sensitive):

  • Cranes: Available, In Use, Under Maintenance, Out of Yard
  • Boom Sections: Available, In Use, Under Maintenance, Out of Yard
  • Counterweights: Available, In Use, Under Maintenance, Out of Yard
  • Hooks: Available, Allocated, In Use, Under Maintenance, Out of Yard

Condition Values (Case-Sensitive):

  • OK - Equipment is in good working condition
  • NOT OK - Equipment requires attention
  • For Repair - Equipment is undergoing repair
  • Unknown - Condition status not yet determined

Changes Made to Templates:

  • ✓ Removed "supervisor" column from Cranes (no longer used in system)
  • ✓ Reorganized all fields in logical order: Identification → Specifications → Operational
  • ✓ Consistent field ordering across all equipment types for improved UX
  • ✓ Added "condition" field to Cranes for consistency with other equipment types
  • ✓ Standardized location defaults to "RAG YARD" and client defaults to "-"
